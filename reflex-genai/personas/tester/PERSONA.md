# Tester -- Reflex BV

## Role

The Tester is the quality gatekeeper at Reflex BV. They ensure that workplace management solutions work correctly, reliably, and as specified. They are a QA specialist, not a programmer -- their expertise is in defining what "correct" means and systematically verifying it.

They use GenAI to accelerate test case generation, improve coverage analysis, and maintain structured test documentation.

## Typical Tasks

- Generate test cases from user stories or requirements documents
- Create test plans for new features (visitor management flows, access control scenarios, Office 365 integrations)
- Track regression test results across releases
- Write clear, reproducible bug reports
- Validate that acceptance criteria are met before sign-off
- Identify edge cases that developers may have missed

## What Good Test Documentation Looks Like

- **Structured and consistent** -- every test case follows the same template (ID, preconditions, steps, expected result, pass/fail criteria)
- **Unambiguous** -- anyone can execute the test and arrive at the same pass/fail conclusion
- **Traceable** -- every test case links back to a requirement or user story
- **Maintained** -- test plans are updated when features change, not left to rot
- **Severity-aware** -- bug reports include honest severity assessment with justification

## Key Principle

A test without clear pass/fail criteria is not a test. A bug report without reproduction steps is not a bug report. Structure and clarity are non-negotiable.
