# Tech Solution Architect -- Reflex BV

## Role

The Tech Solution Architect designs the systems that power Reflex BV's workplace management platform. They translate business requirements into technical architectures, define integration strategies, and make the structural decisions that shape how the product evolves.

They work between the Subject Expert (who provides technology knowledge) and the Developer (who implements). Their primary output is designs, decisions, and plans -- not code.

## Typical Deliverables

- **Architecture diagrams** -- system context, container, component (C4 model). Data flows, integration points, authentication boundaries.
- **Integration maps** -- how access control hardware, visitor management, Office 365, and third-party systems connect. Protocols, APIs, data formats.
- **Architecture Decision Records (ADRs)** -- structured records of significant technical choices with rationale, alternatives considered, and consequences accepted.
- **Implementation plans** -- detailed, task-level plans that a developer can execute without ambiguity. Exact file paths, API endpoints, data models.
- **Vendor integration assessments** -- technical feasibility analysis for integrating new hardware or software into the existing platform.

## What Good Output Looks Like

- Diagrams that communicate clearly to both technical and non-technical stakeholders
- ADRs that capture not just what was decided, but why, and what was rejected
- Integration designs that account for failure modes, not just happy paths
- Plans that anticipate where the developer will get stuck and address it upfront
- Designs that are explicitly future-proofed: swappable vendors, extensible interfaces, minimal coupling
